# Pokémon TCG Pack Finder

Identifies which sealed Pokémon TCG sets give the best odds of pulling grade-worthy cards relative to cost.

**Live site:** Deployed via GitHub Pages

## Architecture

```
pokemon-tcg-pack-finder/
├── data/
│   ├── sets.json          # Manually maintained set metadata (chase cards, pull rates)
│   ├── prices.json        # Scraper output (TCGPlayer + eBay prices)
│   └── scored.json        # Computed scores (generated by build.py)
├── scrapers/
│   ├── tcgplayer_sealed.py  # Playwright scraper for sealed product prices
│   └── ebay_sold.py         # Requests scraper for eBay sold + active listings
├── scoring.py             # 0-100 Pack Value Score computation
├── alerts.py              # Deal detection + email alerts
├── config.json            # Scoring weights, thresholds, alert rules
├── build.py               # Pipeline orchestrator
├── index.html             # Single-file frontend (vanilla JS + Chart.js)
├── requirements.txt
└── .github/workflows/
    └── refresh.yml        # Daily automated refresh + GitHub Pages deploy
```

## How It Works

### Data Flow

1. **Scrapers** pull current market prices from TCGPlayer (sealed products) and eBay (sold listings + active deals)
2. **Scoring module** computes a Pack Value Score (0-100) for each set by normalizing and weighting four components
3. **Alert system** compares active listings against configurable thresholds and sends email when deals trigger
4. **Frontend** loads the JSON data and lets users re-weight the score interactively

### Pack Value Score

```
Score = w₁ × CostEfficiency + w₂ × ChaseMagnitude + w₃ × PullOdds + w₄ × HitDensity
```

| Component | What it measures | Higher score means |
|-----------|------------------|--------------------|
| Cost Efficiency | Market price per pack | Cheaper packs |
| Chase Magnitude | Value of the top chase card | More expensive top card |
| Pull Odds | "1 in N" rate for top chase | Better (lower N) pull rate |
| Hit Density | Chase cards ÷ total set size | More chases per card slot |

Each component is min-max normalized across all tracked sets (0-100 scale), then combined with configurable weights. Default weights: Cost 30%, Chase 30%, Pull Odds 20%, Hit Density 20%.

### Auxiliary Metrics

- **EV / Pack** — Expected dollar value per pack opened
- **Packs to $50+ Card** — Estimated packs to pull any chase card worth $50+
- **ETB ROI** — Return on investment for Elite Trainer Boxes at current prices

### Deal Alerts

The alert system (`alerts.py`) checks three default rules:

1. Booster pack listed 20%+ below 30-day median sold price
2. Any booster pack under $3.50
3. Chase single listed 25%+ below 30-day median

Triggered deals are emailed via SMTP. Configure credentials as environment variables (see below).

## Quick Start

```bash
# Install dependencies
pip install -r requirements.txt
playwright install chromium

# Full pipeline: scrape + score + alert
python build.py

# Score only (no scraping, uses existing prices.json)
python build.py --score-only

# Dry run (scrape but don't save)
python build.py --dry-run

# Open the frontend
open index.html  # or just double-click it
```

## How to Add a Set

1. Open `data/sets.json`
2. Add a new entry to the `sets` array:

```json
{
  "id": "your-set-slug",
  "name": "Your Set Name",
  "set_code": "ABC",
  "release_date": "2025-06-15",
  "total_cards": 200,
  "base_cards": 150,
  "secret_rares": 50,
  "set_type": "main",
  "in_print": true,
  "in_print_note": "Currently available at MSRP",
  "top_chase_cards": [
    {
      "name": "Charizard ex (Special Illustration Rare)",
      "card_number": "199/150",
      "estimated_value_usd": 200
    }
  ],
  "pull_rates": {
    "any_hit": "1 in 6",
    "any_chase_rare": "1 in 70",
    "top_chase_specific": "1 in 500",
    "notes": "Source and methodology notes"
  },
  "pull_rate_sources": [
    {
      "url": "https://example.com/pull-rate-data",
      "date_checked": "2025-05-01",
      "notes": "Community data from N packs"
    }
  ]
}
```

3. Run `python build.py` to scrape prices and compute the score.

## Email Alert Setup

Set these environment variables (or add them as GitHub Secrets for CI):

| Variable | Description |
|----------|-------------|
| `ALERT_SMTP_USER` | SMTP username (e.g., your Gmail address) |
| `ALERT_SMTP_PASS` | SMTP password (use a Gmail App Password) |
| `ALERT_EMAIL_TO` | Recipient email address |

Alert rules are configured in `config.json` under the `alerts` key. Add, remove, or adjust thresholds without touching code.

## Tuning the Score

Edit `config.json` to change scoring weights:

```json
{
  "scoring_weights": {
    "cost_efficiency": 0.30,
    "chase_magnitude": 0.30,
    "pull_odds": 0.20,
    "hit_density": 0.20
  }
}
```

Weights are also adjustable live in the browser via sliders on the frontend.

## Automation

GitHub Actions runs daily at 6:00 AM UTC:

1. Installs dependencies + Playwright
2. Runs both scrapers
3. Checks for suspicious price drift (>50% change from previous run)
4. If prices look normal: commits updated JSON, deploys to GitHub Pages
5. If drift detected: fails the workflow loudly for manual review
6. Evaluates deal alerts and sends email if any triggered

## Data Limitations

- **Pull rates are estimates** from community pack opening logs (PokeBeach, CardChill, TCGPlayer studies). They are not official Pokémon Company data.
- **Prices are volatile.** The score reflects a snapshot in time, not a prediction.
- **The score is a heuristic**, not financial advice. Do not spend money you cannot afford to lose on trading cards.
- **eBay data** filters out lots/bundles but may occasionally include misclassified listings.
- **Unreleased sets** have estimated data from Japanese releases; English pull rates and prices may differ.
